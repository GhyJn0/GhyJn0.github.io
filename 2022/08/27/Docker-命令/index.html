镜像命令

查看镜像

docker images 命令查看所有本地主机上的镜像。

docker images [参数] [镜像[:标签]]
参数
-a :显示所有镜像
-q:显示镜像ID
-aq:显示所有镜像ID

docker images -a

root@ubuntu:/home/test# docker images -a
REPOSITORY      TAG       IMAGE ID       CREATED       SIZE
vulhub/webmin   1.910     8a2b31a69f9a   2 years ago   315MB
vulhub/samba    4.6.3     e37802d9a4e5   4 years ago   622MB

docker images -q

root@ubuntu:/home/test# docker images -q
8a2b31a69f9a
e37802d9a4e5

docker images -aq

root@ubuntu:/home/test# docker images -aq
8a2b31a69f9a
e37802d9a4e5

docker images 镜像名

root@ubuntu:/home/test# docker images vulhub/webmin
REPOSITORY      TAG       IMAGE ID       CREATED       SIZE
vulhub/webmin   1.910     8a2b31a69f9a   2 years ago   315MB
  • REPOSITORY：镜像名（镜像仓库源）。
  • TAG：镜像的标签。
  • IMAGE ID：镜像的 ID。
  • CREATED：镜像的创建时间。
  • SIZE：镜像的大小

运行镜像

docker run

docker run [参数] 镜像名
参数

--name=="名字"：指定容器的名称，如果正在运行该名称的容器，会报错。
--rm：用完即删除，通常用来测试。
-d：后台方式运行。
-it：使用交互方式运行，可以进入容器查看内容。
-e：指定运行环境。
-P：随机指定端口。

-p：指定容器的端口，如：-p 8080:8080。还可以有以下写法：

-p ip:主机端口:容器端口
-p 主机端口:容器端口
-p 容器端口

docker run -it centos /bin/bash #进入容器交互模式

root@ubuntu:/home/test# docker run -it centos /bin/bash
Unable to find image 'centos:latest' locally #检索本地镜像，发现没有该镜像，则去仓库中搜索。

Status: Downloaded newer image for centos:latest

[root@4866999d2709 /]# ls #容器内的centos
bin  dev  etc  home  lib  lib64  lost+found  media  mnt  opt  proc  root  run  sbin  srv  sys  tmp  usr  var
# 由于是以交互方式运行，且进入 /bin/bash 中，此时的路径即为 centos 容器中的 /bin/bash

docker run -d 镜像 #后台启动

test@ubuntu:~$ sudo docker run -d  centos
5e812d3202bb3e63f089edc54d58ec17d07195b407080e28e431ecd8e8cc2df5
test@ubuntu:~$ sudo docker ps 
CONTAINER ID   IMAGE     COMMAND   CREATED   STATUS    PORTS     NAMES

使用 docker run -d 启动，也并不能保证容器一定能在后台运行，如果没有前台使用，容器启动后发现自己没有提供服务，会立刻停止。
