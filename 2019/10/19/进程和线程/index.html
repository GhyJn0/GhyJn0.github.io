  • 执行p1进程，s=1阻塞的4语句可以结束，执行5语句，使用打印机，执行6语句。

  • cpu 执行p0进程，执行1语句，s.value=2，执行完后s.value=1>0,所以执行2语句，使用1号打印机，阻塞队列L中数据Null
  • cpu 执行p1进程，执行4语句，s.value=1, 执行完后s.value=0,所以执行5语句，使用2号打印机,阻塞队列L中数据Null
  • cpu 执行p2进程，执行7语句，s.value=0, 执行一半时s.value=-1<0，所以会执行block 原语，阻塞自己，添加到L阻塞队列,阻塞队列L中数据p2->Null
  • cpu 执行p3进程，执行10语句，s.value=-1,执行一半时，s.value=-2<0,执行block原语，阻塞自己，添加到L阻塞队列。此时L中的数据时p2->p3->Null
  • cpu 执行p0进程(也有可能是p1进程），执行3语句，s.value=-2,执行完后s.value=-1,同时会执行wake唤醒原语，释放1号打印机，开始执行唤醒阻塞队列的p2进程，此时阻塞队列L为p3->Null
  • cpu 执行p2进程，执行8语句，使用1号打印机，然后执行9语句，s.value =-1,执行完后，s.value=0,会执行wake唤醒原语，唤醒p3进程，此时阻塞队列L 为Null,然后释放1号打印机。
  • cpu 执行p3进程，执行11语句，使用1号打印机，然后执行12语句，s.value=0,执行完后s.value=1,然后释放1号打印机，但是此时阻塞队列L为空，不会执行唤醒语句
  • cpu 执行p1进程，执行6号语句，s.value=1,执行完后，s.value=2,然后释放2号打印机

总结

17.读写问题

有读者和写者两组并发进程，共享一个文件，
当两个或以上的读进程同时访问共享数据时
不会产生副作用，但若某个写进程和其他进程（
读进程或写进程）同时访问共享数据时则可能导
致数据不一致的错误。
因此要求：
    ①允许多个读者可以同时对文件执行读操作；
   ②只允许一个写者往文件中写信息；
   ③任一写者在完成写操作之前不允许其他读者或写者工作；
   ④写者执行写操作前，应让已有的读者和写者全部退出。

  • 1.为何两个写进程不能同时访问共享文件（写不能并发执行）
    防止数据覆盖
  • 2.为何读进程和写进程不能同时访问共享文件（读写不能并发）
    防止数据读取的不一致
  • 3.读进程可以同时访问共享文件
    读进程可以并发执行读取

分析存在的关系

  • 互斥
    1.读进程和写进程对共享文件访问
    2.写进程和写进程对共享文件访问
  • 并发
    1.读进程的并发执行

int count =0 ;//表示当前读进程的个数

//存在问题读进程不能并发执行，会出现阻塞
当读者之间存在并发时，p0,p1两个进程，当执行p0时，
第一个进程执行p(rw)后，开始并发执行p1，再一次执
行时发现count==0但是p(rw)回阻塞，导致p1进程必须等
到p0执行完后，才能执行，所以违背读者和写者问题

读者优先

// 解决读者问题不能并发执行
semaphore rw=1; //用于实现对共享文件的互斥访问，表示当前进程访问共享文件
int count =0 ;//表示当前读进程的个数
semaphore mxture=1 //用于表示对count的互斥访问

//写进程
write(){
    while (1)
    {
        p(rw)//写之前加锁
        写文件
        v(rw)//写之后解锁
    }
}


//读进程
read(){
    while(1)
    {
        p(mxture)//用来实现对count的互斥访问
        if(count==0)
            p(rw)//第一个读进程加锁加锁

        count++;//读进程数量
        v(mxture)

        读取文件

        p(mxture)
        count--;
        if (count==0)
            v(rw);//最后一个读进程解锁
        v(mxture)

    }
}

  • 存在两个进程p0,p1当p0先被访问时，p(rw)先上锁，然后执行P1进程，由于此时count!=0所以p1进程不会执行p(rw)，也就不会被阻塞了。实现了并发

  • 问题
    只要读进程在读文件就会导致写进程被堵塞，就会导致饥饿现象
